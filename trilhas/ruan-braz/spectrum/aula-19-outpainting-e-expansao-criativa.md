# Outpainting e expansão criativa

**Tempo estimado de leitura:** 3 minutos

## Objetivos de aprendizado

Ao final desta aula, você será capaz de:

- Aplicar o fluxo de sketch para render usando a ferramenta Viscom e o NanoBanana como alternativa
- Identificar para quais tipos de projetos esse fluxo de prototipagem visual é mais relevante
- Reconhecer quando começar com um desenho manual é mais poderoso do que partir diretamente de um prompt

## O processo de render a partir de rascunhos

A aula apresenta o processo de render na Overlens: quando a equipe faz rascunhos, testa desenhos e quer criar mockups ou representações visuais mais precisas a partir desses esboços. A ferramenta apresentada como referência para esse processo é o Viscom.

O Viscom funciona assim: você joga a sua referência de desenho, coloca o prompt e a ferramenta gera um render respeitando a estrutura do esboço original com precisão. O mesmo desenho que você fez à mão vira uma imagem renderizada, como se fosse uma versão visual polida daquele rascunho. Na aula é mostrado um exemplo onde o mesmo desenho é usado nos dois lados: de um lado o esboço, do outro o render gerado a partir dele. A estrutura é a mesma, só a execução visual muda.

## Para quem esse fluxo é especialmente útil

O professor é direto: para quem trabalha com arquitetura, produto e prototipagem, esse processo é muito poderoso. Você decide a composição, define a estrutura do objeto ou espaço no seu esboço, e a IA executa a visualização. O controle permanece nas mãos do designer ou arquiteto, que decide o que vai onde. A IA só faz o trabalho de renderização.

A Overlens usa esse processo principalmente para exploração e para ensinar. Mas também já começa a usar para os artefatos próprios da plataforma, testando o processo internamente antes de disponibilizá-lo para os alunos.

## Viscom versus NanoBanana: qual usar?

O professor esclarece que o NanoBanana também faz esse mesmo tipo de conversão de sketch para render. A razão pela qual o Viscom é apresentado é histórica: a ferramenta fazia isso antes do NanoBanana ter essa capacidade, então a equipe já tinha um fluxo estabelecido com ela e o professor quis trazer o processo para os alunos. A lógica é a mesma: o mesmo modelo (o rascunho) é retexturizado para ser renderizado e visualizado como imagem final. Não há necessidade de assinar o Viscom se você já usa o NanoBanana para isso.

## Começar desenhando é uma escolha poderosa

A aula termina com uma observação sobre prática na Overlens: a equipe ainda gosta muito de desenhar. Começar com um desenho manual antes de partir para a geração é bom porque clareia na cabeça do criador o que ele quer antes de jogar para a IA derivar. Para worldbuilding, esse processo é especialmente útil: você tem um personagem ou elemento do universo com uma característica específica, começa desenhando para clarear essa ideia e depois usa o desenho como referência para a IA gerar a partir disso.

É um processo muito poderoso e que a equipe gosta de experimentar. A posição do Viscom na sequência de aulas é depois do mood board, dentro da etapa de experimentação.

## Coloque em prática

Faça um esboço simples de um produto, ambiente ou personagem, mesmo que seja um desenho básico. Use esse esboço como referência de entrada no NanoBanana ou no Viscom junto com um prompt descritivo. Observe como a IA preserva a estrutura do desenho e avalie onde o resultado ficou próximo do que você imaginava e onde ele se afastou.
